# MONETIZATION_ROADMAP.md — 有償プランの増やし方

> 原則：**コアの一目（HP/MP）は永久無料**（＝集客とバイラルの燃料）。課金は常に
> 「**深さ（depth）**」と「**広さ（breadth）**」に対して掛ける。プラットフォームの
> 二つの開放端＝**アダプタ（データ）／レンダラ・テーマ（UI）**に沿って商品を増やす。

## 課金の判断ルール（これだけ守る）
- **買い切り**＝価値を一度で届けるもの（テーマ、アダプタ、レンダラ、エフェクト）。
- **継続課金**＝こちらが継続コストを負う／継続価値を出すものだけ（クラウド同期、履歴分析、
  チーム、定期コンテンツ配信）。**コアのウィジェットはサブスク化しない**。
- **無料枠は強く保つ**（単一AI＋基本HUD）。機能の有無ではなく“深さ・広さ”でゲートする。

## 拡張ベクトル（作りやすい順＝資金効率の良い順）
1. **テーマ／コスメ・パック（カタログ型・反復購入）**
   - 季節ドロップ、キャラクター、アニメーションFX、サウンド。制作コスト極小・高頻度。
   - `@includeTheme`構造で“1ファイル＝1商品”。**コミュニティ共作＋レベニューシェア**で量産。
2. **アダプタ・パック（広さ）**
   - 各AIツールが整うたびに単体 or バンドルで販売（Cursor/Copilot/Windsurf/OpenAI/Bedrock…）。
   - レジストリ（`docs/ADAPTERS.md`）に追加するだけ＝原価が低い。Multi-Toolを段階拡張。
3. **レンダラ・パック（UIの広さ）**
   - 配信者キット（OBSアラート/シーン）、メニューバー常駐、VS Codeパネル、モバイル相棒。
   - すべて同じ`state.json`を読むだけ（`docs/RENDERERS.md`）。Web版を足場に横展開。
4. **Pro Analytics（深さ・継続価値）**
   - 履歴・トレンド・コストレポート・バーン予測・「Opusを回す最適時間」提案。
   - ローカル履歴の蓄積＝継続価値 → ここで**低額の継続課金（Pro+）**が正当化される。
5. **Team / Org（高ACV・継続）**
   - 複数席ダッシュボード、共有バジェット、Slack/Discordアラート、管理機能。法人向け。
6. **クラウド同期（任意・継続）**
   - オプトインの履歴ホスティング／デバイス間同期。**ここだけSaaSが妥当**（保管・計算原価）。
   - 既定はローカルファースト（プライバシー＝ブランド）を維持。
7. **マーケットプレイス（長期の本丸）**
   - コミュニティのテーマ／アダプタを販売、QuotaQuestが手数料（take-rate）。
   - プラットフォームが“ストア”になる。最大の複利フライホイール。

## 価格モデルの進化（順番）
```
段階0  買い切り（Gumroad）: Pro $12 / Multi-Tool $24 / Team $99   ← 現状
段階1  テーマ・カタログを反復販売（$3〜9）＋共作レベニューシェア
段階2  Pro+（低額・継続 $3〜5/月）= Analytics＋定期テーマdrop込み
段階3  Team（継続・席課金）= ダッシュボード/アラート/管理
段階4  Cloud同期（任意・継続）＋ Marketplace 手数料（10〜20%）
```
> 継続課金は**段階2以降**。コンテンツ更新と履歴という“続ける理由”が揃ってから入れる。

## トラクション連動（いつ足すか）
- 〜1k DL：買い切りのみ。テーマ2〜3、Multi-Tool 1〜2アダプタで反復購入を作る。
- 1k〜10k：Pro Analytics と Pro+（継続）を投入。配信者キットでバイラル増幅。
- 10k〜：Team／Cloud／Marketplace。コミュニティ供給で原価を外部化。

## 各ベクトルの粗利と狙い
| ベクトル | 原価 | 粗利 | 主目的 |
|---|---|---|---|
| テーマ/コスメ | 極小 | 高 | 反復購入・バイラル |
| アダプタ | 小 | 高 | 広さ（対応AI拡大＝無料層も拡大） |
| レンダラ | 小〜中 | 高 | 新チャネル（配信・モバイル） |
| Analytics | 中 | 高 | 継続課金の正当化 |
| Team | 中 | 高 | 高ACV・安定収益 |
| Cloud | 中（運用） | 中 | 任意・粘着性 |
| Marketplace | 小 | 中〜高 | 複利・供給の外部化 |

## ガードレール
- コア無料を削らない（funnel保護）。広さ・深さで課金。
- サブスク疲れに配慮：買い切り優先、継続は“続ける理由”がある所のみ。
- コミュニティ供給（テーマ/アダプタ）にレベニューシェアで原価を外部化＝最も資金効率が高い。
