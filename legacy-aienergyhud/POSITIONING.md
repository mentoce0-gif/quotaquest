# POSITIONING.md — AI Energy HUD

> Phase 2 / 競合との差別化・ポジショニング
> 最終更新: 2026-06-23

---

## 1. 一行ポジショニング

> **AI Energy HUD は、AI コーディングの「残量不安」を MMORPG の HP/MP ゲージに変える、Windows 常駐のゲーミファイド・オーバーレイである。**

`AI Usage, Gamified.`

---

## 2. ポジショニング・ステートメント（型）

- **For**（誰のために）: Claude Code / Codex / Cursor を毎日使う個人開発者・Vibe Coder・配信者
- **Who**（どんな課題）: 不透明な従量課金で「あと何回叩けるか・いつ回復するか」が体感できず、突然の枠切れや課金急騰に怯えている
- **The AI Energy HUD is**（何か）: デスクトップに常駐する MMORPG 風の HP/MP HUD
- **That**（提供価値）: AI 利用量・回復タイマー・消費速度を、視線を移さず一瞬で把握できる
- **Unlike**（競合と違い）: CLI / Web ダッシュボード / macOS メニューバーに留まる ccusage・tokscale・TokenTracker と異なり
- **Our product**（独自性）: Windows で常時最前面・半透明・配信オーバーレイ対応、かつ **複数 AI を1パーティとして同時に HP 表示** する

---

## 3. 差別化の核（3本柱）

### 柱1: ゲーミフィケーション（体感できる残量）
数字の羅列ではなく **HP/MP ゲージ**。残量が減ると色が変わり（緑→黄→赤）、`STATUS` が `NORMAL / WARNING / CRITICAL` に遷移。RPG の「瀕死」の直感をそのまま使い、見た瞬間に行動を促す。既存ツールが持たない **情動的フィードバック**。

### 柱2: 常駐オーバーレイ（見に行かない）
Rainmeter による常時最前面・半透明。コードを書く視界の端に置けるため、`/status` や `Settings > Usage` を **能動的に開く必要がない**。配信者は OBS にそのままオーバーレイ可能 → 視聴者と残量を共有でき、**配信コンテンツ自体になる**。

### 柱3: パーティ HP（マルチエージェントの同時可視化）★最重要の新規性
Claude / Codex / Cursor を **RPG のパーティメンバーのように横並びで HP 表示**。
- **戦略的背景**: Codex の "Other" 誤計上（Issue #15336）に代表される **サイレントなトークン消費** は、単一ツールの画面だけ見ていると気づけない。複数エージェントを1画面に並べることで「どのメンバーが勝手に MP を吸われているか」が即座に分かる。
- これは ccusage（端末1ツール文脈）や TokenTracker（mac・列挙的）にはない **"パーティ戦況の俯瞰"** という体験。
- 「気づかぬ消費」という痛みが市場で顕在化した今、**パーティ HP は最も刺さる差別化軸**。

## 4. 競合マッピング（2軸）

```
            ゲーム性・常駐性 高
                    ▲
                    │   ★ AI Energy HUD
                    │      (Win/常駐/ゲーム/パーティ)
   TokenTracker ●   │
   (mac widget)     │
                    │
  ───────────────── ┼ ───────────────────►  対応エージェント数 多
                    │            ● tokscale (CLI+leaderboard)
                    │
       公式Usage ●  │   ● ccusage (CLI)
                    │
            ゲーム性・常駐性 低
```

- 右上（ゲーム性×常駐×マルチ）に **空白**。AI Energy HUD が単独で占める。
- ccusage は「正確・軽量だが無味乾燥な CLI」→ **我々は ccusage の出力思想（ローカルJSONL集計）を尊重しつつ、体験レイヤーで差別化**。共存可能（補完関係）。

## 5. 競合別カウンターポジショニング

| 競合 | 相手の強み | 我々の切り返し |
|---|---|---|
| **ccusage** | 高速・正確・OSS定番 | 「ccusage はエンジンに最適。AI Energy HUD はその数字を“見える化”する HUD。両立可能」 |
| **TokenTracker** | 美しい mac ウィジェット | 「Windows ユーザーの唯一解。さらにゲーム性とパーティ表示で体験が一段上」 |
| **tokscale** | リーダーボードで競争性 | 「ソーシャル競争より“自分の戦況把握”。配信オーバーレイで視聴者と共有」 |
| **公式 /status・Usage** | 正確・公式 | 「公式は見に行くもの。我々は見に行かなくても視界に常駐する」 |

## 8. リスクと反証への備え

| 懸念 | 対応 |
|---|---|
| 「数字が見たいだけなら ccusage で十分」 | ターゲットは“体感・常駐・配信・マルチ”を欲する層。数字派は無理に奪わない（共存）。 |
| 「Rainmeter は Windows 限定」 | MVP は意図的に Windows 一点突破（mac は TokenTracker が抑えている＝あえて Windows 空白を取る）。 |
| 「利用量データの取得が壊れやすい」 | ローカル JSONL を読む堅牢方式 + `--mock` フォールバック。ツール側仕様変更はパーサ層で吸収。 |
| 「ゲーム性は一過性」 | コア価値（残量把握・回復予測）は実務。ゲーム性は“入口”として機能。 |

---

## 9. 結論

AI Energy HUD は「**Windows × ゲームUI × 常駐オーバーレイ × マルチエージェントのパーティ HP**」という、現存ツールが誰も占めていない座標に立つ。とりわけ **パーティ HP によるサイレント消費の可視化** は、2026年の課金不透明化・Codex 誤計上問題という追い風に直接乗るため、これを第一の旗印に据える。
