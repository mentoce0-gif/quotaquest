FAQ

The honest answers.

Does it need my API key or send data anywhere?

No. It only reads the local JSONL usage logs that Claude Code and Codex already write to your disk (~/.claude, ~/.codex), computes the gauges locally, and writes a small state.json the skin renders. Zero network calls, zero telemetry.

How is this different from ccusage / TokenTracker?

Those are excellent CLIs / dashboards / a macOS menu-bar app. AI Energy HUD is the only Windows, always-on, gamified desktop overlay with multi-agent party HP. It complements ccusage — you can run both.

The limits aren't exact tokens — does the HUD still work?

Yes. Anthropic/OpenAI report limits relatively, so you set hpMaxTokens to the budget a full bar should represent. It's intentionally calibratable so a vendor wording change never breaks it.

What's "Party HP" really for?

Codex has a known issue where background tokens are misclassified as "Other" and silently drain your weekly cap. Watching one tool's screen won't catch it — a party view of all your agents will. Unknown token data shows as ?? UNKNOWN rather than a false-safe full bar.

Do I need to be a Rainmeter expert?

No. Run install.bat — it copies the skin, seeds the config, and starts the monitor. Drag the HUD where you like; right-click for options.

Mac / Linux?

v1.0 is Windows-first (Rainmeter). A cross-platform companion is on the roadmap. macOS users today are well served by TokenTracker.
